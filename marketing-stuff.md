Para o Argos, eu iria com um 60/30/10 bem orientado a produto analítico moderno — limpo, técnico e escalável para dashboard.

## Paleta recomendada

### 60% — Base neutra

Fundo, superfícies, cards, áreas de leitura.

- `#F5F7FA` → light background principal
- `#E9EEF5` → superfícies secundárias
- `#1A1F2B` → dark mode base

Isso mantém o sistema respirando e deixa os dados dominarem a interface.

---

### 30% — Cor principal (brand)

A identidade do Argos.

- `#0A2A66` → azul profundo principal
- `#123D91` → hover/ativo
- `#3B82F6` → highlights suaves

Esse azul passa:

- confiabilidade
- vigilância
- inteligência
- infraestrutura

Sem cair no azul corporativo morto estilo banco/SaaS genérico.

---

### 10% — Cor de acento

Onde você chama atenção:

- execuções ativas
- indicadores
- insights
- CTA
- gráficos prioritários

Minha recomendação:

#### Opção A (mais premium)

- `#00C2A8` → teal/cyan analítico

Fica sofisticado com o azul e combina perfeitamente com monitoramento.

#### Opção B (mais agressiva)

- `#FFB020` → amber

Excelente para dashboards operacionais e alertas.

---

## Distribuição visual real

### Interface

- 60% → fundos/cards/tabelas
- 30% → navbar, ícones, botões principais
- 10% → estados vivos, métricas, gráficos, badges

## O erro que você precisa evitar

Não transforme o Argos num carnaval visual de dashboard.

Seu produto é:

- observabilidade
- coleta
- inteligência documental
- rastreabilidade

Então a UI precisa transmitir:

- precisão
- silêncio visual
- densidade informacional controlada

Muito neon, muito gradiente, muito glow:
→ mata credibilidade imediatamente.

O logo já aponta para uma direção mais séria e cirúrgica.
A interface inteira deveria seguir essa mesma linguagem.
